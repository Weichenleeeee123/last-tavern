# 【生活娱乐】临终酒馆 - 与历史人物对饮最后一杯

## 0. 自我介绍

大家好，我是 Weichen Li，一名热爱前端开发的全栈开发者。平时喜欢用 React + Tailwind 做一些有创意的小项目，这次参赛希望用 AI + 前端技术创造一个有温度、有沉浸感的体验。

## 1. Demo 简介

**临终酒馆** 是一个沉浸式 Web 体验——你推开一扇古老的酒馆大门，走进一间烛光摇曳的酒馆，20位历史人物正坐在各自的桌前，每个人都在他们人生的最后一夜。你选择一位，入座对饮，听他讲述自己的故事，试图改变他的命运——但历史自有引力。

- **产品形态**：单页沉浸式 Web 应用（React + Vite + Tailwind CSS）
- **面向谁**：对历史感兴趣、喜欢深度对话体验的用户
- **核心功能**：
  1. **推门入场**：逼真的双开门 3D 推开动画，进入暗黑氛围酒馆
  2. **20位历史人物围桌而坐**：6张主题桌（权力/笔墨/镜中/真理/音符/星辰），每桌人物围坐一圈，俯视椭圆木桌带蜡烛和酒杯
  3. **角色主动开口**：进入对话后角色会先自我介绍生平背景，然后开始对饮
  4. **历史引力机制**：用户尝试说服角色改变命运，角色会有情感波动但最终回归史实
  5. **引导选项**：每次角色回复后生成3个可选回应，降低对话门槛
  6. **对饮记录卡**：结束对话后生成羊皮纸风格的记录卡，可保存为图片

## 2. Demo 创作思路

**灵感来源**：有一次我在想，如果能穿越回过去，在那些伟大人物生命的最后一晚和他们喝一杯，会聊些什么？项羽会不会后悔不过江东？梵高在开枪前在想什么？爱因斯坦在拒绝手术时说了什么？这些问题驱使我做了这个项目。

**想解决的问题**：市面上的 AI 对话产品大多是"你问我答"的工具感，缺乏情感沉浸和场景感。临终酒馆试图把 AI 对话包装成一个有仪式感的体验——你不是在跟 AI 聊天，你是在和一个人的灵魂对饮。

**为什么做这个方向**：历史人物对话天然适合 LLM 的角色扮演能力，而"最后一夜"的设定天然制造了戏剧张力。加上"历史引力"机制，让对话不只是闲聊，而是一次有意义的尝试——你能说服历史吗？

## 4. TRAE 实践过程

### 技术栈
- React 19 + Vite 8 + TypeScript + Tailwind CSS 3
- Zustand 5（状态管理 + localStorage 持久化）
- Web Audio API（程序化音效：门声、酒杯、钟声）
- html2canvas（记录卡截图导出）

### 开发流程

**第一阶段：创意设计**
用 TRAE 的头脑风暴功能，从"LLM + 历史人物对话"出发，逐步明确：
- 13位→20位人物，覆盖政治、文学、科学、音乐、艺术等领域
- "历史引力"机制：用户劝说推进"改变"条，角色回应拉回"命运"条
- 单页沉浸式架构，4个场景：门→酒馆大厅→对话浮层→记录卡

**第二阶段：核心实现**
1. **角色系统**：20位历史人物，每人有系统提示词（身份锚定+历史约束+对话规范+回复格式）、生平背景、临终独白、史料锚点
2. **命运之秤**：用户每次发言 +15 倾向改变，角色回复 -20 拉回命运，最多12轮
3. **对话持久化**：切换角色自动保存/恢复对话记录，刷新不丢失
4. **角色主动开口**：进入对话自动触发角色介绍生平

**第三阶段：视觉打磨**
1. AI 生成20张统一油画风格肖像 + 3张氛围背景图
2. 真实双开门（3D 透视 + 木纹 + 铁铰链 + 铆钉 + 光缝）
3. 椭圆木桌（木纹年轮 + 划痕 + 蜡烛 + 酒杯 + 地面投影）
4. 对话浮层（backdrop blur 虚化酒馆背景，聚焦当前角色）
5. 羊皮纸记录卡（深色文字在浅色背景上清晰可读）

**第四阶段：音效与体验优化**
1. Web Audio API 程序化生成5种音效（门推、酒杯、发送、接收、背景音乐）
2. 引导选项：LLM 每次回复后生成3个可选回应
3. 移动端适配
